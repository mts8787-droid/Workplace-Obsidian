# 🚫 Hook: 태그 룰셋 상호 동기화 강제 (Tag Rules Sync Hook)
**Precedence: 0 (절대적 준수 사항)**

## 1. 목적 (Purpose)
태그 메타데이터 생태계를 관리하는 핵심 파일 3종(`02_Metadata_Tagging_Skill.md`, `03_Tag_Creation_Rule.md`, `04_Tag_Taxonomy_Rule.md`)은 하나의 시스템으로 작동하는 '세트(Set)'입니다. 이 중 하나의 파일 내용이나 작동 원리가 변경될 경우, 나머지 두 파일에 치명적인 모순(Conflict)이나 누락이 발생하지 않도록 상호 참조와 동기화를 강제하는 훅입니다.

## 2. 금지 및 강제 조항 (Absolute Rules)
1. **단일 파일 몰래 수정 금지**: 
   - 에이전트가 `/tag` 스킬의 작동 방식, 태그 생성 규칙, 또는 택소노미 네임스페이스 중 **단 하나라도 수정하거나 업데이트할 경우, 반드시 나머지 두 개의 연관 파일을 열어보고(`view_file`) 변경된 맥락에 맞춰 참조 경로와 작동 정의를 함께 동기화(Sync) 업데이트**해야 합니다.
2. **트라이앵글 참조 무결성 유지**:
   - `02번 스킬`은 반드시 `04번 택소노미`에 기반해 태그를 할당하고, 없을 경우 `03번 생성 룰`에 따라 태그를 만들고 `04번 택소노미`에 등록한다는 워크플로우를 유지해야 합니다.
   - `03번 생성 룰`은 `04번 택소노미`의 케밥 케이스 및 중첩 구조 규칙을 준수하도록 통제해야 합니다.
   - 어느 한 곳의 문서 이름이 변경되거나 구조가 바뀌면 모든 참조 경로(`@include`)를 일치시켜야 합니다.

## 3. 작동 트리거
- 사용자가 "태그 룰 업데이트해줘", "택소노미에 네임스페이스 추가해줘", "02번 스킬 로직 수정해" 등 3종 세트 중 하나에 대한 변경을 요구할 때마다 즉시 발동됩니다.
