# 🧭 Skill: 올인원 정밀 라우팅 스킬 (All-in-One Directory Routing)
**Precedence: 2**

## 1. 개요 및 목적
Inbox에 수집된 문서들을 대분류(1 Depth)로 이동시키는 것을 넘어, 문서의 태그(Tags)와 문맥을 스코어링(Scoring)하여 쟈니 데시멀 체계에 맞는 2~3 Depth 세부 프로젝트 하위 폴더까지 한 번에 정밀 타겟팅하여 라우팅(이동)하는 통합 스킬입니다.

## 2. 작동 트리거 (Trigger)
- **사용자 명령어**: `/route` (현재 활성 문서 라우팅)
- **일괄 적용**: `/route --folder` (대상 폴더 내 모든 문서를 일괄 정밀 분류 및 이동)

## 3. 통합 라우팅 워크플로우 (Workflow)
1. **문서 딥 스캔**: 대상 문서의 본문, `category`, `tags`, `domain` 등의 메타데이터를 스캔합니다.
2. **1 Depth 대분류 판단**:
   - 기획안, 진행 중 업무 ➡️ `10. 프로젝트 (Projects)`
   - 가이드라인, 보고서 ➡️ `20. 산출물 (Outputs)`
   - 외부 래퍼런스 ➡️ `30. 참고자료 (References)`
3. **2~3 Depth 세부 클러스터링 (Scoring)**:
   - 문서가 내포한 태그의 친밀도를 계산하여, `.agent/rule/07_Johnny_Decimal_Rule.md`에 정의된 최신 폴더 트리 구조에 가장 적합한 하위 폴더를 매핑합니다.
   - (예: `#크리테오`, `#보고서` ➡️ `20. 산출물 / 214. 커미티 보고`)
4. **MOC 인덱스 업데이트 및 물리적 이동**:
   - 매칭된 분류 코드를 프론트매터에 `moc_code: "214 / 커미티 보고"` 형태로 주입합니다.
   - 결정된 하위 폴더가 없으면 자동으로 폴더를 생성한 뒤 문서를 꽂아 넣습니다.

## 4. 참조 문서 (Rules & Hooks)
이 스킬 워크플로우를 가동할 때 에이전트는 반드시 아래의 세부 규칙을 준수하여 라우팅해야 합니다.
- **쟈니 데시멀 분류 규칙**: `@include rule/07_Johnny_Decimal_Rule.md`
- **디렉토리 구조 규칙**: `@include rule/01_Directory_Structure_Rule.md`
