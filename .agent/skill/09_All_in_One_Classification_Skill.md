# 🛠️ Skill: 올인원 문서 분류 스킬 (/hotkey)
**Precedence: 1**

## 1. 목적 (Purpose)
기존에 개별적으로 동작하던 태깅, 제목 최적화, 링크 연결, MOC(목차) 갱신 스킬을 하나의 파이프라인으로 통합하여, 문서를 지식 저장소(Vault)에 완벽하게 정착시키는 "원터치 올인원 분류"를 수행합니다.

## 2. 작동 조건 (Triggers)
- **대상 파일**: 정규화 및 분류가 필요한 마크다운(`.md`) 문서
- **사용자 명령어**: 
  - `/hotkey`: 현재 포커스된 단일 문서에 대해 통합 파이프라인을 1회 실행
  - `/hotkey --folder [경로]`: 지정된 폴더 내의 파일들을 대상으로 파이프라인 일괄 실행

> [!WARNING] HOOK: 폴더 단위 실행 시 순차 처리 원칙
> `--folder` 전역 변수(옵션)가 포함된 경우, 폴더 내의 여러 파일을 절대 동시에 병렬로 처리하지 마십시오. **반드시 파일 1개를 선택하여 아래 Step 1~4를 모두 완료하고 저장을 끝낸 후, 다음 파일로 넘어가는 순차적 직렬 처리(Sequential Processing)**를 엄격히 준수해야 합니다. 

## 3. 통합 파이프라인 워크플로우 (Sequential Workflow)

본 스킬이 발동되면, AI는 대상 문서 1개에 대해 아래 4개의 기존 스킬 로직을 **순서대로 누락 없이** 적용합니다.

### Step 1: 메타데이터 및 태깅 부여 (Ref. `02_Metadata_Tagging_Skill`)
- 문서 본문을 분석하여 옵시디언 표준 프론트매터(15개 속성)를 삽입합니다.
- 문서 속성과 문맥에 맞는 수평적 컨텍스트 태그(`#태그`)를 본문이나 프론트매터에 부여합니다.

### Step 2: 제목 최적화 (Ref. `06_Title_Change_Skill`)
- 문서의 핵심 내용과 부여된 메타데이터를 바탕으로 직관적이고 표준화된 제목(파일명)으로 변경합니다.
- 파일명이 변경될 경우, 이 문서를 참조하고 있는 다른 문서들의 백링크를 안전하게 찾아내어 업데이트합니다.

### Step 3: 문맥 링크 연결 및 고립 해소 (Ref. `04_Orphan_Linker_Skill`)
- 문서 내부의 주요 키워드를 분석하여 Vault 내의 다른 관련 문서로 이어지는 양방향 링크(`[[문서명]]`)를 본문에 자연스럽게 삽입합니다.
- 문서가 Vault 내에서 고립(Orphan)되지 않도록 지식망에 연결합니다.

### Step 4: 쟈니 데시멀 정밀 라우팅 (Ref. `01_Directory_Routing_Skill`)
- 문서의 최종 제목과 링크가 확정되면, 해당 문서가 속하는 카테고리나 주제를 딥러닝 스코어링하여 `.agent/rule/07_Johnny_Decimal_Rule.md` 트리에 명시된 2~3 Depth 세부 하위 폴더로 자동 라우팅(이동)시킵니다.

## 4. 완료 및 보고
- 단일 파일 처리가 끝날 때마다 위 4단계가 성공적으로 적용되었는지 점검합니다.
- `--folder` 모드일 경우, 한 파일의 처리가 끝날 때마다 사용자에게 어떤 작업들이 수행되었는지 간략히 요약 보고하고 다음 파일로 넘어갑니다.
