# 🛠️ Skill: MD-PPT 파싱 및 정밀 구조화 스킬 (/md-ppt)
**Precedence: 2**

## 1. 목적
이 스킬은 PPTX 또는 PPT 기반 추출 문서를 읽어 들여 텍스트, 표, 차트, 다이어그램, 강조 요소를 의미 있는 Markdown 구조로 복원한다.

## 2. 동작 조건 (Triggers)
- **대상 파일**: `00. Inbox`에 있는 PPTX 원본, 또는 사용자가 현재 열어 둔 변환 대상 문서
- **명령어**
  - `/md-ppt [파일명 또는 경로]`: 특정 대상 문서에 대해 4-Agent 파싱을 실행한다. 예: `/md-ppt FFFFFFFFFFF.pptx`
  - `/md-ppt --folder [경로]`: 대기열의 문서를 대상으로 순차 파싱을 시작한다.
  - `/md-ppt`: 경로가 없으면 IDE의 현재 활성 탭을 먼저 확인하고, 그 파일이 변환 대상이면 사용자에게 확인 질문을 보낸 뒤 실행한다. 활성 파일이 없거나 대상이 아니면 경로를 다시 요청한다.
- **묻기 규칙 (Ask First)**: 사용자가 경로를 명시하지 않고 `/md-ppt`만 입력한 경우, 에이전트는 현재 열린 파일을 우선 경로로 잡아 변환 여부를 묻는다. 사용자 승인이 있어야만 실행하고, 열린 파일을 사용할 수 없을 때만 타겟 파일명이나 경로를 요청한다.

## 3. 참조 규칙
- `@include rule/08_PPT_Parsing_Rule.md`
- `@include hook/03_PPT_Parsing_Hook.md`

## 4. 처리 파이프라인
이 스킬은 다음 순서로 동작한다.

1. **Parser 단계**: `python -X utf8 ".agent/scripts/autonomous_ppt_agent.py" "<PPTX 경로>"`를 사용해 텍스트와 슬라이드 이미지를 1차 추출한다.
2. **Omni-Agent 단계**: 추출 결과를 읽고 500줄 단위로 검토하면서 표, Mermaid, callout, 강조 텍스트를 정리한다.
3. **정리 단계**: 원본의 파편화된 구조를 다시 결합하고, 필요하면 표나 Mermaid로 재구성한다.

### 4.1 전제 조건
- `pywin32`와 `win32com`이 설치되어 있어야 슬라이드 이미지를 함께 추출할 수 있다.
- 이미지 추출이 불가능해도 텍스트 파싱은 계속 진행할 수 있다.
- PPT 파싱은 `hook/03_PPT_Parsing_Hook.md`의 제약을 따른다.

### 4.2 실행 원칙
- 한 번에 처리하는 대상은 최대 2개 문서로 제한한다.
- 중간 결과만 남기고 끝내지 않는다.
- 파싱이 끝나면 최종 마크다운이 사용자에게 바로 활용 가능한 형태인지 확인한다.

### 4.3 채팅 기반 실행
- 사용자가 `/md-ppt [파일명]` 형태로 명시하면, 에이전트는 대상 파일의 절대 경로를 찾아 자동 실행한다.
- 사용자가 경로 없이 `/md-ppt`만 입력하면, 에이전트는 현재 활성 탭의 파일 경로를 우선 사용해 사용자에게 확인한 뒤 실행한다.
- 활성 탭이 없거나 변환 대상이 아니면, 그때 경로를 요청한다.

## 5. 실행 후 기대 결과
- 변환된 `.md` 파일
- 필요한 경우 `assets/` 아래 슬라이드 이미지
- 표, 다이어그램, 차트가 정리된 읽기 쉬운 Markdown 문서
