# Skill: MD-Excel 정밀 변환 및 정제 (/md-xlsx)
**Precedence: 2**

## 1. 목적
`/md-xlsx`은 Excel 데이터의 특성(Sheet 분리, 병합 셀, 간트 차트 형태의 공백 열 등)을 파악하여, 깨지기 쉬운 거대한 원시 표를 가독성 높은 논리적 마크다운 표와 리스트로 재구성합니다. 데이터를 단순 복사하는 것을 넘어, 빈칸으로 유실된 문맥(병합 셀)을 복원하고 시트별 인사이트를 구조화합니다.

## 2. 작동 조건
- **대상 파일**: XLSX, CSV, XLS 등 스프레드시트 파일
- **명령어**
  - `/md-xlsx [파일명 또는 경로]`
  - `/md-xlsx`만 입력된 경우 현재 활성 파일을 확인하고, 없으면 경로를 요청합니다.

## 3. 참조 규칙

- `@include rule/06_Obsidian_Syntax_Rule.md`

## 4. 처리 파이프라인
1. **Parser 단계**: 시트(Sheet) 단위로 데이터를 추출합니다. 이때 마크다운 표의 구조가 깨지는 것을 방지하기 위해 셀 내부의 줄바꿈(`\n`)은 반드시 `<br>` 태그로 치환합니다.
2. **Structure 단계**: 시트명과 컬럼 헤더(`Unnamed: X` 등)를 분석하여 데이터의 성격(예: 종합 현황, 개별 과제, 타임라인 등)을 분류합니다.
3. **LLM 정제 및 문맥 복원 단계**: (아래 5번 원칙 적용)
4. **Inbox 정착 단계**: 완성된 마크다운 결과물을 `00. Inbox` 폴더에 저장하고, 즉시 `/taggging`과 `/title` 스킬을 자동 실행하여 메타데이터를 부여합니다.

## 5. Excel 특화 데이터 정밀 구조화 원칙 (LLM Self-Correction)

원시 Excel 추출물은 `Unnamed: X` 헤더, 무의미한 빈칸(`|||`), 텅 빈 주차별 간트 차트 구조로 인해 가독성이 매우 떨어집니다. LLM은 이를 극복하기 위해 아래 원칙을 수행합니다.

- **병합 셀 문맥 복원 (Fill-down & Unpivot)**: 
  - 병합된 셀이 풀리면서 생긴 수많은 빈칸(`|||`)이나 `Unnamed` 컬럼은 상위 카테고리나 프로젝트 그룹을 의미합니다.
  - LLM은 위아래 문맥을 추론하여 비어있는 셀에 상위 대분류를 명시적으로 채워 넣습니다. 특히 **'과제명'**이나 **'결과지표(KPI)'**와 같은 핵심 메타데이터는 표 내부에 가두지 않고, 마크다운 리스트 최상단에 별도 고정 배치하여 문맥을 명확히 합니다.
- **가로 간트 차트의 세로 리스트화 (Timeline 압축)**: 
  - 주차별(예: `4/1W`, `4/2W`...)로 길게 나열된 간트 차트형 컬럼 중, 일정이 존재하지 않는 텅 빈 열(`||||`)은 표에서 모두 삭제합니다.
  - 가로로 비정상적으로 넓은 표는 가독성이 매우 떨어지므로, 일정이 존재하는 주차의 데이터만 추출하여 **세로형 타임라인 표(주차 | 항목 | 내용)** 형식으로 구조를 완전히 재조립합니다.
- **시트(Sheet) 간 데이터 중복 제거 및 논리적 통합**:
  - '종합' 시트와 '과제별' 시트에 동일한 데이터가 중복되어 존재할 경우, 종합 시트는 '요약(Summary)' 관점으로 가볍게 서술하고, 세부 과제 시트에서 상세 테이블을 전개하여 중복을 피합니다.
- **셀 내 다중 텍스트 및 기호의 안전한 구조화 (`<br>` 치환)**:
  - 하나의 셀 안에 불릿포인트(`-`, `□`)가 섞여 길게 작성된 텍스트는 마크다운 표 문법(`|`)과 충돌하여 표 전체를 붕괴시킵니다.
  - 이를 방지하기 위해 셀 내부의 모든 줄바꿈(`\n`)은 반드시 `<br>` 태그로 자동 치환하여, 한 칸 안에 텍스트가 깔끔하게 정렬되도록 보호합니다.

## 6. 실행 제약
- 한 번에 변환하는 시트의 수가 10개를 초과할 경우, 가장 핵심적인 요약 시트 3~4개를 우선 정제하고 나머지는 원본 데이터를 유지하여 토큰 초과를 방지합니다.
