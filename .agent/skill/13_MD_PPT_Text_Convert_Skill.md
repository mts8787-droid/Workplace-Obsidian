# Skill: MD-PPT 텍스트 기반 정밀 구조화 (/md-ppt-text)
**Precedence: 2**

## 1. 목적
`/md-ppt-text`는 토큰 소모가 큰 이미지 파싱을 제외하고, PPTX/PPT 문서의 텍스트 레이어만 빠르고 정밀하게 읽어 슬라이드의 의미와 구조를 파악합니다. 이미지 렌더링 없이도 텍스트의 맥락을 분석하여 LLM이 슬라이드마다 흩어진 단어들을 자연스러운 문장과 불릿포인트, 표 형식으로 재정리하여 가독성 높은 Markdown 문서를 완성합니다.

Mermaid는 `10_Mermaid_Rendering_Rule.md`의 조건을 엄격히 준수하여 생성합니다.

## 2. 작동 조건
- **대상 파일**: PPTX, PPT, 또는 사용자가 명시한 PPT 변환 대상 문서
- **명령어**
  - `/md-ppt-text [파일명 또는 경로]`
  - `/md-ppt-text --folder [경로]`
  - `/md-ppt-text`만 입력된 경우 현재 IDE 활성 파일을 우선 확인하고, 변환 대상이 아니면 경로를 다시 요청합니다.

## 3. 참조 규칙
아래 문서는 `/md-ppt-text` 수행 시 항상 함께 적용합니다.

- `@include rule/08_PPT_Parsing_Rule.md`
- `@include rule/10_Mermaid_Rendering_Rule.md`
- `@include rule/06_Obsidian_Syntax_Rule.md`
- `@include hook/03_PPT_Parsing_Hook.md`

## 4. 처리 파이프라인
1. **Parser 단계**: 문서의 텍스트 레이어만 빠르게 추출합니다. (이미지/시각적 렌더링 파싱 완전 생략)
2. **Structure 단계**: 추출된 텍스트 계층과 들여쓰기를 분석하여 표, 다이어그램, 계층형 리스트로 분류합니다.
3. **Markdown 재구성 단계 (LLM 정제 - 축약 금지)**: 
   - 파편화된 원시 텍스트 덤프를 단순히 결론만 요약(축약)하지 않고, **텍스트의 맥락, 배경, 슬라이드의 논리 구조를 원본 그대로 보존**하여 풀어서 작성합니다.
   - 특히 **로드맵 일정(월/년 단위), 기대효과, ROI 수치, 예산, 절감액, 구체적인 에이전트/솔루션 이름, Pain Point** 등 구체적인 데이터는 절대로 생략하거나 뭉뚱그리지 않고 표와 불릿포인트로 원본과 동일한 정밀도로 복원합니다.
4. **Mermaid 방어 단계**: `10_Mermaid_Rendering_Rule.md`의 정제 규칙을 적용합니다.
5. **Reviewer 단계**: Mermaid 렌더링 실패 조건, 깨진 YAML, 깨진 fence를 전수 검사합니다.
6. **Inbox 정착 단계**: 완성된 마크다운 결과물을 `00. Inbox` 폴더에 저장(이동)하고, 즉시 `/taggging`과 `/title` 스킬을 순차적으로 자동 실행하여 파일 최적화를 완료합니다.

## 5. 텍스트 기반 시각화 및 정밀 구조화 원칙 (Self-Correction & Reassembly)
이미지 렌더링을 생략한 원시 추출물은 텍스트 파편화, 문맥 단절, 위계 상실 등의 치명적인 문제점을 지닙니다. LLM은 이를 극복하기 위해 아래의 **'능동적 문맥 재조립 및 오류 보정'** 원칙을 수행해야 합니다.

- **표/다이어그램 주변 파편 텍스트의 문맥적 재결합 (Contextual Re-attachment)**: 추출 과정에서 표(Table) 외부로 밀려나거나 동떨어진 불릿포인트로 나열된 라벨, 날짜, 범례, 헤더(예: `MAaMAS vs HSAD`, `12/9-12/21`)를 능동적으로 파악하여, 연관된 표의 제목이나 적절한 셀 내부로 문맥에 맞게 재결합합니다.
- **도식 공간 분석(`[도식 공간 분석]`) 텍스트의 논리적 추론 및 흐름 복원**: 좌표 기반으로 무작위 나열된 다이어그램 텍스트 파편(예: 1. PMax, 2. DV360, 3. CM360)을 단순히 나열하지 않습니다. LLM의 도메인 지식과 슬라이드 문맥을 바탕으로 파편들 간의 관계(인과관계, 선후관계, AS-IS vs TO-BE 등)를 추론하여 의미 있는 파이프라인 흐름이나 마크다운 표로 완벽히 재구성합니다.
- **방향성 기호 및 무의미한 불릿 문자(`|`, `•`, `☛`, ``)의 지능적 해석**: 원시 텍스트에 단독으로 존재하는 `- |` 또는 `` 기호는 단순 오타가 아니라 프로세스 흐름(Flow)을 나타내는 화살표(A → B)로 해석하여 파이프라인을 구축합니다. 반면, `- •` 처럼 의미 없이 텍스트로 추출된 빈 불릿 기호는 렌더링 시 깔끔하게 삭제(필터링)합니다.
- **슬라이드 간 연속성(Cross-slide) 및 누락 데이터 방어**: `(뒷장 계속)`, `(2/2)` 등의 표현이 있으나 다음 슬라이드에 표 텍스트가 없는 경우(이미지로만 삽입되어 파싱이 생략된 경우), 내용을 지어내지 않고 `[이미지 객체로 포함되어 텍스트 추출 생략됨]` 이라는 명확한 주석을 달아 혼선을 방지합니다. 이어지는 텍스트가 존재할 경우 하나의 표/리스트로 병합합니다.
- **초정밀 문장화 및 시각적 위계(Parent-Child)의 의미론적 복원**: 단순히 결과만 요약하지 않습니다. 평면적으로 나열된(Flattened) 텍스트 덤프들을 의미론적으로 분석하여, 대분류(Parent)-소분류(Child)의 위계를 마크다운 중첩 리스트로 명확히 복원하고 자연스러운 문장으로 재조립합니다.
- **수치 및 데이터 테이블화 (Table)**: 투자비용, 기대효과(ROI), 로드맵 타임라인 등 수치 데이터와 병렬 구조의 텍스트는 원본 데이터의 누락 없이 마크다운 표 형식으로 정밀하게 묶어 가독성을 높입니다.
- 방향성이 명확하게 유추되는 텍스트 흐름은 조건부로 Mermaid를 사용합니다.

## 6. 실행 제약
- 한 번에 처리하는 대상은 최대 2개 문서로 제한합니다.
- 추출 과정 중 텍스트의 유실이 의심되더라도 추측성 내용 생성은 금지하며, 있는 텍스트 기반으로만 도식화합니다.
