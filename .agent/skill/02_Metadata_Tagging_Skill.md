# 🏷️ Skill: 메타데이터 및 다차원 택소노미 태깅 (/tag)
**Precedence: 2**

## 1. 목적 (Purpose)
문서의 본문 맥락(Context)을 심층 분석하여 상세 설명(Description) 및 15개 필수 프론트매터를 정밀하게 채워 넣습니다. 더불어 단순 키워드 추출을 넘어, **15개의 강력한 중첩 네임스페이스 기반의 태그 택소노미(Tag Taxonomy)**에 맞추어 태그를 할당하거나 룰에 따라 신규 태그를 생성 및 등록합니다.

## 2. 작동 트리거 (Trigger)
- **명령어**: `/tag` (현재 문서 대상 정밀 태깅)
- **일괄 적용**: `/tag --folder [경로]` (대상 폴더 내 모든 문서의 메타데이터 및 태그 일괄 최적화)

## 3. 참조 문서 (Rules & Hooks)
이 스킬을 가동할 때 에이전트는 **태그 관리 3종 세트**의 룰과 훅을 반드시 연계하여 준수해야 합니다.
- **동기화 강제 훅**: `@include hook/05_Tag_Sync_Hook.md` (02번 스킬, 03번 생성 룰, 04번 택소노미 룰의 상호 동기화 강제)
- **태그 택소노미 룰**: `@include rule/04_Tag_Taxonomy_Rule.md` (15개 네임스페이스 분류 기준 및 기존 단어장)
- **태그 생성 룰**: `@include rule/03_Tag_Creation_Rule.md` (신규 태그 생성 시 케밥 케이스 및 사전 자동 업데이트 원칙)

## 4. 🤖 작동 방식 및 워크플로우 (Workflow)
1. **문서 맥락 분석**:
   - `view_file`로 타겟 마크다운 문서의 본문을 읽어 들여 핵심 도메인, 연관 부서, 주요 제품, 마케팅 액션 등을 파악합니다.
2. **택소노미 기반 태그 매핑**:
   - 분석된 맥락을 바탕으로 `04_Tag_Taxonomy_Rule.md` 문서를 조회하여, 이미 등록된 15개 네임스페이스(`#type`, `#market`, `#product`, `#topic` 등) 하위의 값 중 가장 적합한 태그들을 추출 및 할당합니다.
3. **신규 태그 생성 및 등록**:
   - 만약 본문의 핵심 개념을 표현할 적절한 기존 태그가 사전에 없다면, `03_Tag_Creation_Rule.md`에 의거하여 **새로운 태그를 생성**합니다. (예: 새로운 광고 플랫폼 등장 시 `#ad/new-platform` 생성)
   - **(핵심)** 생성된 신규 태그는 문서에 삽입됨과 동시에, 에이전트가 즉시 `04_Tag_Taxonomy_Rule.md` 파일에 접근해 해당 네임스페이스 리스트에 **신규 값을 영구 등록(업데이트)**해야 합니다.
4. **위키링크 전환**:
   - 특정 인물이나 거래처, 특정 모델명 등 고유 개체는 태그로 달지 않고 본문에 위키링크(`[[ ]]`) 처리합니다.
