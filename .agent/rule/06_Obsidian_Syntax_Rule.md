# ✍️ Rule: 옵시디언 특화 문법 규칙 (Obsidian Syntax Rule)
**Precedence: 2**

## 1. 개요
에이전트가 문서를 작성하거나 아키텍처(다이어그램) 설계를 수행할 때, 사용자의 옵시디언(Obsidian) 환경에서 파싱(Parsing) 에러나 렌더링 깨짐이 발생하지 않도록 강제하는 문법 규칙입니다.

## 2. 다이어그램 (Mermaid) 규칙
옵시디언의 내장 Mermaid 렌더러는 특수문자 처리에 매우 민감하므로 다음 규칙을 절대 엄수합니다.
- **라벨 따옴표 처리**: 노드(Node) 라벨에 괄호`()`, 대괄호`[]`, 슬래시`/`, 띄어쓰기 등 특수문자가 포함될 경우 파싱 에러를 막기 위해 반드시 큰따옴표로 전체 라벨을 감싸야 합니다.
  - ❌ 잘못된 예: `id[Label (Extra)]`
  - ✅ 올바른 예: `id["Label (Extra)"]`
- **HTML 태그 금지**: 다이어그램 라벨 내부에는 `<br>`, `<b>` 등의 HTML 태그 사용을 절대 금지합니다.

## 3. 참조 방식 구분: `@include` vs `[[wikilink]]`
볼트 내에서 문서 간 연결 시, '에이전트용'과 '사람용'을 명확히 구분하여 사용합니다.
- **에이전트 시스템 인클루드 (`@path`)**: 
  - 에이전트가 시스템 프롬프트(컨텍스트)로 읽어들여야 하는 시스템 파일이나 규칙 파일 연결에는 `@include 파일경로.md` 형태를 사용합니다.
- **인간 열람용 양방향 링크 (`[[name]]`)**: 
  - 사용자가 지식망을 탐색하고 클릭해서 이동하기 위한 지식(일반 노트) 연결 목적일 경우, 반드시 옵시디언 고유 문법인 대괄호 `[[문서명]]` 형식을 사용합니다.

## 4. 콜아웃(Callout) 적극 활용
옵시디언에서는 중요한 정보나 경고를 일반 텍스트 볼드체보다 콜아웃(Callout) 블록으로 표현하는 것이 가독성이 좋습니다. 에이전트는 요약이나 주의사항 작성 시 이를 적극 활용합니다.
- 예: `> [!note]`, `> [!warning]`, `> [!info]`, `> [!important]`
