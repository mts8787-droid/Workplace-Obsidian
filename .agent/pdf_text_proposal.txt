2개 World Model의 역할 - DualMinds
SWM: 온톨로지/월드 DSL로 개념과 관계를 정의하고, 상태(State), 전이(Transition), 제약(Constraint), 단일 진실 원천
(SSOT), 최종 확정(Execution & Commit)으로 구성되고, 세계를 판정 가능하고 집행 가능한 형태로 고정
NWM: LLM을 코어로 삼아 관측(Perception), 근거 고정(Grounding), 잠재 상태 추정(Belief), 전개 예측(Dynamics), 액션 후보 
생성(Planner)하며, 열린 현실을 해석하고 의미를 구성
FSM (유한 상태 머신) 구성요소
FSM = Shared State Store + Agent Dispatcher
FSM은 월드 모델과 여러 구성요소가 겹쳐 페어링되는 실행엔진입니다.

MAaMAS 실행은 ‘추론’이 아니라 ‘프로토콜 기반의 집행 절차’로 굴러갑니다
User Query
 → 허용된 Transition 후보를 World States에서 선택
 → Guard(정책/제약) 통과 시에만
 → Action 실행 → Event 발생
 → Trigger → Agent Activation → Agent Action Request
 → World Model Confirms & Permits → Next State + Audit Log 업데이트
이 구조가 에이전틱 자동화 뿐 아니라
운영 안정성/확장성/감사 가능성을 구조적으로 보장합니다.FSM이 에이전트들을 “자동 소집”하는 방식
사람이 각 에이전트를 시키는 게 아니라, FSM이 이벤트를 받으면 필요한 
에이전트를 자동 호출합니다
● FSM은 Event를 받으면, 현재 State에서 가능한 Transition 후보를 고릅니다.
● 그 Transition을 완성하기 위해 필요한 역할(agents)을 매핑합니다.
● 각 에이전트는 역할 단위로 작업 수행 → 결과를 공유 상태/로그(SSOT)에 
반영
● FSM이 결과를 취합해 Next State로 전이합니다.
